BEHAVIORAL PATTERNS:
---------------------

1. Memento pattern
  - The Memento pattern is a behavioral design pattern that allows an object to capture its internal 
  state and restore it later without exposing its implementation details. 
  It is useful for implementing undo/redo functionality in applications.
  - The Memento pattern is used to restore an object to a previous state.


2. State pattern
  - The State pattern is a behavioral design pattern that allows an object to change its behavior 
  when its internal state changes. It is useful for implementing finite state machines and 
  managing complex state transitions in applications.
  - The State pattern is used to manage the state of an object and change its behavior based on that state.
  - lot of if/else conditions can be replaced with state pattern. It is used to manage the state of an object and 
  change its behavior based on that state.


3. Strategy pattern
  - The Strategy pattern is a behavioral design pattern that allows an object to change its behavior 
  at runtime by encapsulating different algorithms or strategies and making them interchangeable.
  - The Strategy pattern is used to define a family of algorithms, encapsulate each one, and make them 
  interchangeable. It allows the algorithm to vary independently from clients that use it.


4. Iterator pattern
  - The Iterator pattern provides a way of iterating over an object without having to expose the object's internal
  structure, which may change in the future. Changing the internals of an object should not affect its 
  consumers.

5. Command pattern
  - The Command pattern is a behavioral design pattern that encapsulates a request as an object, 
  allowing you to parameterize clients with different requests, queue or log requests, and support undoable operations.
  - The Command pattern is used to encapsulate a request as an object, allowing you to parameterize clients with 
  different requests, queue or log requests, and support undoable operations.
  - Command pattern breaks the app into layers, following good software design principle called "principle of 
    separation of concerns". 

6. Template Method pattern
  - The Template Method pattern is a behavioral design pattern that defines the skeleton of an algorithm 
  in a base class, allowing subclasses to provide specific implementations for certain steps of the algorithm.
  - The Template Method pattern is used to define the skeleton of an algorithm in a base class, allowing 
  subclasses to provide specific implementations for certain steps of the algorithm.

7. Observer pattern
  - The Observer pattern is a behavioral design pattern that allows an object (the subject) to notify 
  other objects (the observers) when its state changes, without the subject needing to know the details of the observers.
  - The Observer pattern is used to establish a one-to-many dependency between objects, so that when one object changes 
  state, all its dependents are notified and updated automatically.

8. Mediator pattern
  - The Mediator pattern is a behavioral design pattern that defines an object that encapsulates how a set of objects 
  interact, promoting loose coupling between them. It allows objects to communicate with each other through a mediator 
  object, rather than directly.
  - The Mediator pattern is used to reduce the complexity of communication between multiple objects by centralizing 
  the communication logic in a mediator object.
